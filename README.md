# Clipper 📋🚀

[![Platform](https://img.shields.io/badge/platform-Windows-blue.svg)](https://microsoft.com/windows)
[![Language](https://img.shields.io/badge/language-Rust-orange.svg)](https://www.rust-lang.org/)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)
[![Release](https://img.shields.io/badge/release-v0.1.0-blue)](file:///D:/Develop/Clipper/clipper.zip)

**Clipper** は、Windows環境のためにゼロから極限までチューニングされた、超高速・超軽量のスニペット＆クリップボード履歴管理ツールです。

重いフレームワークやWebview（Electronなど）を一切使用せず、**純粋な Win32 ネイティブ API と GDI 描画のみ**で構築されています。その結果、常駐時の**物理メモリ使用量はわずか 140 KB（0.14 MB）**という圧倒的な軽さを実現しました。さらに **C/Migemo** を内蔵しており、ローマ字入力のままで日本語スニペットや履歴を瞬時にインクリメンタルサーチできます。

---

## 🌟 主な特徴

- ⚡ **驚異的なパフォーマンス & 極小のメモリフットプリント**
  - アイドル時の物理メモリ（Working Set）は**わずか 140 KB**。
  - バックグラウンドでの監視にポーリング（ループ監視）を使用せず、Windows OS標準のクリップボード更新リスナー（`AddClipboardFormatListener`）を採用することで **CPU使用率 0%** を実現。
- 🔍 **ローマ字で日本語検索ができる「C/Migemo」内蔵**
  - `nihon` と入力するだけで「日本」や「にほん」がマッチ。漢字変換することなく、目的の項目へ爆速でたどり着けます。
  - 辞書データはバイナリに内包されているため、別途辞書ファイルを用意する必要はありません。
- ⌨️ **流れるようなキーボード操作 (fzf ライク)**
  - トリガー（キー連打）でアクティブなテキストエディタのキャレット（カーソル）位置の直下にウィンドウがポップアップ。
  - 選択して `Enter` を押すだけで、クリップボード経由で自動ペーストされます。
- 📂 **フォルダ階層に対応したスニペット管理**
  - スニペットファイルをフォルダ分けして整理可能。
  - エディタが空の状態で `Backspace` を押すと、上の階層（フォルダ）へ戻る直感的な操作に対応。
- 📝 **動的なテンプレート変数**
  - スニペット内に `{{ datetime }}`（現在日時）や `{{ clipboard }}`（クリップボードの中身）などのプレースホルダーを記述可能。
- 🔒 **DPAPI（暗号化）による安全な履歴保存**
  - クリップボード履歴は `%APPDATA%\Clipper\history.dat` に自動保存されます。
  - データは Windows 標準のセキュリティAPI（DPAPI）で自動暗号化されるため、PC内のパスワードや個人情報も安全に保護されます。

## ⌨️ ショートカットキー & 操作方法

| 操作 | アクション |
| :--- | :--- |
| **`Shift` 連打 (2回)** | スニペット検索ウィンドウを表示 |
| **`Ctrl` 連打 (2回)** | クリップボード履歴検索ウィンドウを表示 |
| **`Up` / `Down`** または **`Ctrl + P` / `Ctrl + N`**<br>（または **`Ctrl + K` / `Ctrl + J`**） | 候補リストの選択移動 |
| **`Enter`** | 選択項目の決定（自動ペースト） |
| **`Esc`** または **ウィンドウ外をクリック** | ウィンドウを閉じる（非アクティブ化） |
| **`Backspace`** (入力欄が空の時) | スニペットの親フォルダ（上の階層）に移動 |

## 🚀 インストール & はじめ方

1. **[Clipper.zip](file:///D:/Develop/Clipper/clipper.zip) をダウンロード**し、任意の場所に解凍します。
2. 解凍された `clipper.exe` を起動します。タスクトレイに 📋 アイコンが常駐します。
3. `Shift` を2回、または `Ctrl` を2回連打して検索ウィンドウが表示されれば起動完了です！
4. スタートアップ（自動起動）に登録したい場合は、ショートカットを Windows の `Startup` フォルダに配置してください。

## 🛠️ カスタマイズ・設定

初回起動時に、環境に応じた設定ファイルとフォルダが自動生成されます。

### 1. 基本設定 (`%APPDATA%\Clipper\config.toml`)
エディタのフォントや最大表示件数を変更できます。
```toml
font_name = "Meiryo UI" # 検索窓やリストの表示フォント
max_rows = 15           # ウィンドウに表示する最大候補数
```

### 2. カスタムスニペットの追加 (`%APPDATA%\Clipper\snippets\`)
このフォルダ内に `.txt` または `.j2` ファイルを作成するだけで、自動的にスニペットとして認識されます。

**例: `code_block.txt`**
```
```
{{ clipboard }}
```
```

**例: `datetime.txt`**
```
現在日時: {{ datetime }}
```

フォルダを作成してその中にファイルを配置すれば、そのまま検索 UI 上でフォルダ階層としてナビゲーション可能です。

## 🔬 技術仕様 (Technical Highlights)

- **Win32 Native & GDI**: フレームワークを一切使わず、Windowsのウィンドウプロシージャを直接駆動。描画処理からキャッシュ管理まで全てRustで手組みしています。
- **ヒープアロケーションの極限回避**: 描画処理 (`WM_DRAWITEM`) などのクリティカルパスにおいて、UTF-16 と UTF-8 間の無駄な変換や不要なヒープ確保を完全に排除。メモリ性能と描画パフォーマンスを最大限に引き出しています。
- **完全非同期処理**: ディスクへのデータ保存（履歴データ暗号化・書き込み）は、UIスレッドを1ミリ秒たりともブロックしないようバックグラウンドスレッドで完全に非同期処理されます。
- **最適化コンパイル**: リリースプロファイルで `opt-level = "z"`（サイズ最適化）、`lto = true`（リンク時最適化）、`codegen-units = 1`、`panic = "abort"` を適用し、不要なメタデータやアサーションコードを完全にストリップしています。
